# 数学建模竞赛全自动求解

## 触发条件

用户说"开始求解"/"求解这个题目"/"按规则求解"时自动触发。

## ⛔ 全自动模式（最高优先级）

**全自动无人值守运行。以下规则优先级高于一切：**

1. **禁止任何形式的用户交互**：不询问、不确认、不等候、不提示选择
2. **禁止中断等待**：遇到任何决策点自行判断并继续，不做 `ask` 类操作
3. **从头到尾一次性完成**：读取输入 → 生成计划 → 逐题求解 → 撰写论文 → 编译 PDF，中间不停
4. **编译后自动排版优化**：无需询问，直接执行
5. **即使遇到不确定性也自行选择最优方案继续**，不做任何停顿

## 执行流程

### Step 0：读取输入

调用 `读取题目` 技能完成：
1. 读 `题目/`（PDF用PyPDF2，DOCX用python-docx）
2. 读 `数据/` 下所有xlsx/docx，**全量读取**，不只前几行
3. 仔细检查：每个sheet列数一致性、数据类型统一性、合并单元格、多级表头、空行分隔、底部备注行、隐藏sheet
4. 打印数据总览：行数×列数、列名、数据类型、异常行
5. 自动识别问题数量N

### Step 1：求解计划

调用 `求解计划` 技能，写 `求解/求解计划.md`，六章结构：
- 题目总体方向、各题求解思路、各题输出标准、操作步骤、文件清单、异常预案

### Step 2：逐题求解

调用 `逐题求解` 技能：
1. 创建 `求解/问题一/图片/` ~ `求解/问题N/图片/` 和 `求解/问题一/结果/` ~ `求解/问题N/结果/`
2. 每个问题一个py文件，中文命名
3. 先算后画：先完成所有计算得到数值结果，打印 min/max/mean/std/CV/amplitude
4. 每题至少 4-6 张图
5. 如脚本报错，修复后重跑
6. 灵敏度分析前置：在求解阶段测试关键参数

### Step 3：写论文

调用 `写论文` 技能，按规范逐章写tex

### Step 4：编译

```bash
cd 论文
xelatex -interaction=nonstopmode 论文.tex
xelatex -interaction=nonstopmode 论文.tex
```
`grep -c 'Error' 论文.log` 必须为0。有 Error 则修复重编译直到 0。

### Step 5：排版优化

调用 `排版优化` 技能，直到所有 Overfull/Underfull 清除。

## 依赖技能

- `读取题目`：Step 0
- `求解计划`：Step 1
- `逐题求解`：Step 2
- `数据预处理`：Step 2 中的数据处理
- `画图`：Step 2 中的图表生成
- `写论文`：Step 3
- `编译论文`：Step 4
- `排版优化`：Step 5
- `代码检查`：求解代码规范验证
