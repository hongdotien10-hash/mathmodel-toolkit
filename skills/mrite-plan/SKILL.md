# 求解计划

## 目的

读完题目和数据后，生成 `求解/求解计划.md`，规划六章结构，写完直接开干。

## 计划结构

### 一、题目总体方向（一整段）

以文章风格写出，包含：
- 问题类型分析（优化/预测/评价/分类/机理分析）
- 核心任务提炼
- 数据规模与特征概览
- 各问题之间的递进逻辑关系

### 二、各题求解思路

每个子问题包含三个部分，文章风格，不分点：

**具体分析**：输入是什么？输出是什么？难点在哪？问题定位是什么类型？

**建模方案**：
- 核心公式推导方向
- 选择什么方法？为什么？
- 备选方法是什么？

**方法匹配表**（每问末尾附一个表格）：

| 维度 | 主方法 | 备选方法 | 关键算法/工具 |
|------|--------|----------|---------------|
| 数据预处理 | ... | ... | ... |
| 特征工程 | ... | ... | ... |
| 核心模型 | ... | ... | ... |
| 求解算法 | ... | ... | ... |
| 结果验证 | ... | ... | ... |

### 三、各题输出标准（含图表规划）

每问必须产出的：
- 图表清单（类型+描述+预期发现）
- 表格清单（内容+列数）
- 数值结果清单（关键输出值+解释）

### 四、操作步骤（含依赖顺序）

每问从加载→预处理→建模→求解→输出的完整链路：
```
问题一：
  1. 加载 数据/xxx.xlsx Sheet2
  2. 预处理：缺失值填充 → 异常值检测 → 标准化
  3. 输出 求解/预处理数据.csv（供后续问题共用）
  4. 建模：...
  5. 求解：...
  6. 输出：图X张 + 表X个 + 结果CSV X个

问题二（依赖问题一的预处理数据.csv）：
  1. 加载 求解/预处理数据.csv
  ...
```

```
...
```

### 六、异常预案

| 异常情况 | 备选处理 |
|----------|----------|
| 数据大量缺失（>20%） | 改用多重插补或删除该特征 |
| 模型不收敛 | 调整学习率/正则化参数/换初始值 |
| 数据量不足 | 数据增强/交叉验证/bootstrap |
| 结果不合理 | 检查预处理是否正确/换备选模型 |
| 编译报错 | 检查tex语法/字体/包的兼容性 |

## 输出

写入 `求解/求解计划.md`，不需要用户确认，直接进入下一阶段求解。
