# 2022电工杯B题 建模分析报告

## 赛题概述

**题目**：5G网络下的应急物资配送问题 — 车辆+无人机协同路径优化

**本赛题共4个子问题。**

### 关键参数（题面给定）

| 参数 | 值 |
|------|-----|
| 车辆平均速度 | 50 km/h |
| 无人机平均速度 | 75 km/h |
| 无人机最长飞行时间 | 70 min |
| 无人机最大载重 | 50 kg |
| 每中心配置 | 1辆车 + 1架无人机 |
| Q1/Q2 车辆容量 | 1000 kg |
| Q3/Q4 车辆容量 | 500 kg |
| 出发点 (Q1-Q3) | 地点9 |

## 子问题拆解

### 问题一：纯车辆配送（14地点, 1000kg）

**输入**：附件1 — 14个地点的稀疏路网距离矩阵（48条边/196可能, 24.5%填充率）。路网不完全连通。

**决策变量**：车辆访问地点的顺序（TSP回路）。

**目标函数**：最小化总配送距离（等价于最小化总配送时间，速度恒定50km/h）。

**约束条件**：
1. 每个地点恰好被访问一次
2. 车辆总载重 ≤ 1000kg
3. 从地点9出发，最终返回地点9（或任意点结束，题面说"返回集散点的时间为最后一次配送完成时间"）

**数学本质**：带容量约束的车辆路径问题(CVRP)，但总需求<容量则为TSP。

**求解策略**：
1. Floyd-Warshall：稀疏图 → 全对最短路径完全图
2. TSP启发式：多起点最近邻(NN) + 2-opt局部搜索 + 模拟退火(SA)
3. 配送时间 = 总距离 / 50 km/h

**预期结果**：最短路径约582km，配送时间约6.32h。

**图表**：TSP最优回路图（标注地点序号和距离）、路径距离对比表

### 问题二：车辆+无人机协同配送（14地点, 1000kg）

**输入**：附件2 — 14地点路网。实线表示车辆+无人机均可通行，虚线表示仅无人机通行。61条边。

**决策变量**：
1. 车辆行驶路线
2. 无人机每次飞行的起降点、目标地点、飞行路线
3. 车辆与无人机的会合时间点

**目标函数**：最小化总配送完成时间。

**约束条件**：
1. 每个地点需求被满足
2. 无人机单次飞行 ≤ 70分钟
3. 无人机载重 ≤ 50kg
4. 车辆载重 ≤ 1000kg
5. 无人机必须在车辆处起降（或可在地点等待）
6. 车辆和无人机路线的时间协调

**数学本质**：车辆-无人机协同路径问题(VRPD/TSP-D)，NP-hard。

**求解策略**：
1. Floyd-Warshall 分别计算车辆路网和无人机路网的全对最短路径
2. 两阶段启发式：
   - Phase 1：TSP求解纯车辆路线
   - Phase 2：贪心分配无人机任务 — 扫描路线上每个地点，若无人机可从当前车辆位置飞到目标地点并在70min内返回会合点，则分配给无人机
3. 无人机任务分配以总时间最小为目标

**图表**：车辆+无人机协同路线图（不同颜色标注）、任务分配表

### 问题四：选址+路径联合优化（30地点, 500kg）

**输入**：附件2的数据扩展到30个地点（需要从附件2获取30×30矩阵）。需从30个地点中选择2个作为应急物资集中点。

**决策变量**：
1. 2个集中点的位置（从30个候选地点中选择）
2. 每个集中点服务的区域划分
3. 每个集中点的车辆+无人机配送路线

**目标函数**：最小化两个集中点的总配送完成时间（取max）。

**约束条件**：
1. 恰好选2个集中点
2. 每个普通地点分配到恰好1个集中点
3. 每个集中点配置：1车+1无人机，车容量500kg
4. 无人机约束同Q2/Q3

**数学本质**：选址-路径问题(LRP) + 车辆-无人机协同。双层优化。

**求解策略**：
1. 外层：枚举/启发式选择2个集中点（30选2=435种组合，可枚举）
2. 内层：对每种选址方案，将30个地点分配到最近的集中点（Voronoi划分）
3. 对每个集中点的子问题调用Q2/Q3求解器
4. 选总时间最小的选址方案

**图表**：最优选址地图（标注两个集中点和分区）、各方案对比表

## 数据理解

### 附件1（Q1）

- 15行×15列（1行表头 + 14地点距离矩阵）
- 48条有向边，对称矩阵
- 距离范围：18-61（单位未标注，应为km，因为与答案582km吻合）
- 稀疏度：24.5%
- 地点9为出发点

### 附件2（Q2/Q3/Q4）

- 结构同上但边数更多（61条）
- 包含实线边（车辆+无人机）和虚线边（仅无人机）
- 需区分两类边

### 数据问题

1. 附件2未明确区分实线/虚线 — 需从题面图2读取边类型
2. 地点的物资需求量未在附件中明确 — 可能需要从题面或额外数据获取
3. 30个地点的数据在附件2中可能不全 — 当前只读到14×14

## 假设敏感性预检

### 模糊表述及解释

1. **"总配送时间"定义**：题面说"返回集散点的时间为最后一次配送完成时间"。理解为车辆最终返回集散点的时间。

2. **无人机能否独立起飞**：题面说"无人机也可被无人机从某地点接回" → 无人机可从车辆当前位置起飞，完成配送后返回车辆位置。

3. **地点需求是否超出容量**：若所有地点需求总和<车辆容量，Q1退化为TSP。582km的结果暗示总需求<1000kg（单车足够）。

### 最终采用的解释

- Q1为TSP（总需求<容量），单辆车一次回路即可
- 无人机任务独立分配，不改变车辆主线
- 速度恒定，不考虑装卸时间
- 路网对称（无向图）
